# docs

끼리 프로젝트의 개발 과정을 기록하는 폴더입니다.

```
docs/
├── README.md       ← 지금 이 파일
├── decisions.md    ← 기술 선택 의사결정
├── dev-log.md      ← 개발 기록
└── trouble.md      ← 트러블슈팅
```

각 파일에 날짜 기준으로 위에서 아래로 누적합니다.  
매일 안 써도 괜찮고, 쓸 것이 생겼을 때만 추가하면 됩니다.

---

## decisions.md 템플릿

```markdown
## YYYY-MM-DD · 제목

**결정**: 무엇을 선택했는지 한 줄.

**이유**
- 

**검토한 대안**
- A: 제외한 이유
- B: 제외한 이유
```

---

## dev-log.md 템플릿

```markdown
## YYYY-MM-DD

**한 것**
- 

**막힌 것**
- 

**다음에 할 것**
- 
```

---

## trouble.md 템플릿

```markdown
## YYYY-MM-DD · 이슈 제목

**증상**: 어떤 문제가 발생했는지  
**원인**: 왜 발생했는지  
**해결**: 어떻게 고쳤는지  
**배운 점**: 다음엔 어떻게 할지
```
